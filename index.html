QueryDoc — RAG Platform

View on GitHub
Project Overview

Generative AI is powerful, but it hallucinates. When accuracy is paramount, relying on a model's pre-trained memory isn't enough.

QueryDoc bridges this gap. By leveraging Retrieval-Augmented Generation (RAG), it allows users to chat directly with their own documents. Instead of generic answers, QueryDoc extracts precise information grounded in the uploaded text, providing transparent source citations for every claim.

Architecture
Document Upload ↓ Text Chunking & Tokenization ↓ Embedding Generation (OpenAI / HuggingFace) ↓ Vector Store Indexing (FAISS) ↓ User Query Input ↓ Semantic Search & Context Retrieval ↓ LLM Prompt Generation with Context ↓ Grounded Response with Citations
Key Features
  • End-to-end RAG pipeline for custom documents
  • PDF and DOCX automatic text extraction
  • Semantic vector search using FAISS
  • Context-aware conversational memory
  • Adjustable retrieval parameters (chunk size, overlap)
  • Source citation for hallucination reduction

AI-Powered Industrial Vision

View on GitHub
System Demonstration

AI-Powered Industrial Vision Inspection System is a real-time machine vision simulation that replicates an industrial conveyor belt inspection station. Combines Unity for the digital twin simulation, FastAPI for backend communication, and OpenCV for computer vision-based inspection.

Packages travel along a conveyor belt where an industrial inspection camera captures images. These images are sent to a Python FastAPI server, where OpenCV analyzes the package and classifies it as PASS or FAIL. Defective packages are automatically rejected via a simulated reject mechanism.

Architecture
Package Spawn ↓ Conveyor Belt ↓ Industrial Inspection Camera ↓ RenderTexture Image Capture ↓ FastAPI Server ↓ OpenCV Image Processing ↓ PASS / FAIL Classification ↓ Inspection Result Sent Back to Unity ↓ Status Indicator ↓ Automatic Reject Mechanism
Key Features
  • Real-time conveyor belt simulation
  • Industrial inspection camera simulation
  • Image capture using Unity RenderTexture
  • Computer vision inspection using OpenCV
  • FastAPI REST API integration
  • Real-time Unity ↔ Python communication
  • Automatic PASS / FAIL classification
  • Automated reject mechanism for defective products
  • Live inspection status visualization
  • Modular architecture for future AI model integration
